# 🚀 갓생 RPG (Grow-Up)

> “나의 하루가 내 캐릭터의 경험치가 된다.”

---

# 🛠️ 로컬 실행 가이드

## 실행 전 준비

- Java 17
- MySQL 8.x
- IntelliJ 또는 Spring Boot 실행 가능한 IDE

## 실행 순서

1. MySQL에서 프로젝트용 DB를 생성합니다.

2. 루트 경로의 `.env.example`를 참고해서 `.env` 파일을 생성합니다.

3. 애플리케이션을 실행하면 Flyway가 자동으로 초기 스키마를 반영합니다.
4. 
4. 기본 접속 경로는 `http://localhost:8080` 입니다.

## 팀원 작업 시작 전 체크

- `.env` 파일이 정상 작성되어 있는지 확인
- 로컬 MySQL이 실행 중인지 확인
- 브랜치는 `feat/{issue번호}-설명` 형식으로 생성
- 작업 전에 `main` 최신 상태를 기준으로 브랜치를 분기

# 📌 프로젝트 소개

갓생 RPG는 사용자가 하루 목표를 직접 등록하고,  
실제 수행 인증을 통해 경험치를 획득하며 캐릭터를 성장시키는 성장형 자기관리 서비스입니다.

단순 TODO 앱이 아니라,

- 미션 등록
- 사진 인증
- EXP 성장
- 레벨업
- 랭킹 경쟁
- 공개 프로필

을 통해 게임처럼 동기부여를 제공하는 것이 핵심입니다.

---

# 🎯 핵심 유저 플로우

```text
회원가입 → 오늘 미션 등록 → 사진 인증 → 경험치 획득 → 레벨업 → 랭킹 경쟁
```

## 전체 흐름

1. 회원가입 / 로그인
2. 오늘의 미션 등록
    - 하 1개
    - 중 1개
    - 상 1개
3. 미션 수행 후 사진 인증
4. 인증 완료 즉시 DONE 처리 + EXP 지급
5. 경험치 누적으로 레벨업
6. 랭킹 및 공개 프로필 확인

---

# 🧩 핵심 기능

## ✅ 회원가입 / 로그인

- 아이디 + 비밀번호 기반 로그인
- Spring Security 세션 인증 방식 사용
- BCrypt 비밀번호 암호화
- 로그인하지 않은 사용자는 `/login`으로 리다이렉트

---

## ✅ 오늘의 미션 등록

사용자는 하루에 총 3개의 미션만 등록 가능

| 난이도 | 제한 |
| --- | --- |
| 상 | 1개 |
| 중 | 1개 |
| 하 | 1개 |

### 등록 규칙

- 같은 난이도 중복 등록 불가
- 하루 최대 3개
- 오늘 날짜 기준으로만 등록 가능

---

## ✅ 사진 인증

미션 수행 후 사진 업로드를 통해 인증

### 인증 방식

- `multipart/form-data`
- JPG / PNG 업로드 가능
- 업로드 즉시 인증 완료 처리
- 별도 관리자 검수 없음

### 인증 가이드

1. 미션 수행 확인이 가능한 사진을 올려주세요.
2. 사진은 선명하고 알아볼 수 있게 올려주세요.
3. 미션과 관련없는 사진 업로드는 자제해주세요.

---

## ✅ 성장 시스템

미션 완료 즉시 경험치 획득

| 난이도 | EXP |
| --- | --- |
| 하 | +10 |
| 중 | +30 |
| 상 | +50 |

### 하루 최대 획득 가능 EXP

```text
10 + 30 + 50 = 최대 90 EXP
```

---

## ✅ 레벨업 시스템

### 레벨업 공식

```text
요구 EXP = 100 + (현재 레벨 - 1) * 10
```

### 예시

| 레벨 | 필요 EXP |
| --- | --- |
| Lv1 → Lv2 | 100 |
| Lv2 → Lv3 | 110 |
| Lv3 → Lv4 | 120 |

### 초과 경험치 이월

```text
Lv1 EXP 120 달성
→ Lv2로 레벨업
→ 남은 20 EXP 유지
```

### 프로그레스 바 계산

```text
현재 EXP / 요구 EXP * 100%
```

---

## ✅ 랭킹 시스템

전체 유저 기준 랭킹 제공

### 정렬 기준

```text
1순위: 레벨 내림차순
2순위: EXP 내림차순
```

### 메인 화면

- TOP 5 미리보기 제공

### 랭킹 상세 페이지

- 전체 유저 조회
- 페이지네이션 적용
- 10명 단위 조회

---

## ✅ 공개 프로필

닉네임 클릭 시 공개 프로필 진입 가능

### 공개 정보

- 닉네임
- 현재 레벨
- 현재 EXP
- 완료된 미션 목록
- 인증 사진

### 비공개 정책

- TODO 상태 미션은 공개되지 않음
- DONE 상태 미션만 공개

### 표시 방식

- 날짜별로 그룹화하여 표시

예시:

```text
2026-05-18
- 상: 헬스장 가기
- 중: 아침 식사 챙겨먹기
- 하: 물 2L 마시기
```

# 📦 패키지 구조 가이드

기본 방향은 `기능별 패키지 분리`입니다. 공통 코드는 `global`, 기능 코드는 각 도메인 패키지 안에서 관리합니다.

```text
com.godsaeng.godsaeng_up
├─ global
├─ home
├─ auth
├─ mission
├─ profile
└─ ranking
```

## 기능별 담당 기준

- `auth`: 회원가입, 로그인, Security 연동
- `mission`: 미션 등록, 조회, 수정, 상태 처리
- `profile`: 공개 프로필, 레벨/경험치 표시
- `ranking`: 랭킹 조회, TOP 5, 페이지네이션
- `home`: 온보딩(`/`), 메인 대시보드(`/main`)

## 패키지 내부 권장 구조

기능 패키지 내부에서는 아래 구조를 기본으로 사용합니다.

```text
feature
├─ controller
├─ service
├─ repository
├─ entity
└─ dto
```

예시:

```text
mission
├─ controller
├─ service
├─ repository
├─ entity
└─ dto
```
